# DSA Roadmap for This Repo

This roadmap turns the repo's lecture flow into a practical study order.
Use it when you want to revise from basics to interview-style problems without guessing what to do next.

## Phase 1: Orientation and Problem-Solving Mindset

Lectures: 1-3

What to focus on:
- Understanding how the course is structured
- Learning what an algorithm is
- Practicing logical-building questions
- Converting problems into flowcharts and pseudocode before coding

Core ideas:
- Step-by-step thinking
- Breaking a problem into small actions
- Explaining logic before writing syntax

Questions to revise:
- Chocolate Wrapper Exchange
- Minimum Cuts for a 7-Unit Gold Bar
- Measure 9 Minutes with 4 and 7 Minute Hourglasses
- Bridge and Torch Crossing
- Camel and Bananas

Outcome:
- You should be able to explain a solution in words before writing code.

## Phase 2: Basic Programming Foundations

Lectures: 4, 6, 12

What to focus on:
- Input and output
- if-else
- loops
- data types
- functions and function calls

Core ideas:
- Condition-based branching
- Repetition with for and while loops
- Using helper functions to reduce repeated logic

Questions to revise:
- Add Two Numbers
- Simple Interest Calculation
- Check Whether a Number is Odd or Even
- Maximum of Two Numbers
- Maximum of Three Numbers
- Grade Card Program
- Print Counting from 1 to N
- Sum of N Natural Numbers
- Factorial of a Number
- Lottery Game
- Lottery Game Part 2

Outcome:
- You should be comfortable writing small C++ programs without needing a template every time.

## Phase 3: Pattern Printing and Nested Loops

Lectures: 5, 7, 8, 9, 10, 11, 14, 15, 17

What to focus on:
- Nested loops
- Row and column thinking
- Space handling
- Symmetry and diagonals
- Number-based patterns

Core ideas:
- Outer loop controls rows
- Inner loop controls columns
- Printing often depends on a condition involving i, j, n, or spaces
- Trace a pattern row by row before coding

Questions to revise:
- Pattern 1-6 across lecture files
- Row Symmetric Pattern 1
- Row Asymmetric Pattern 2
- Column Symmetric Pattern 3
- Column Asymmetric Pattern 4
- Extra Border and Diagonals Pattern
- Number Diamond Mirror Pattern
- Fibonacci Pattern
- Pascal Triangle Pattern
- Swastik Pattern

Outcome:
- You should be able to identify whether a pattern is space-driven, symmetry-driven, or math-driven.

## Phase 4: Math, Digits, and Number Theory

Lectures: 13, 14, 15, 20, 23, 24

What to focus on:
- gcd
- prime checks
- digit extraction
- reverse and sum of digits
- base conversion
- special number problems

Core ideas:
- Use `% 10` and `/ 10` for digit work
- Use `i * i <= n` for efficient factor checking
- Convert through decimal for base-conversion problems
- Separate helper functions for cleaner math code

Questions to revise:
- GCD Brute Force
- GCD Reverse Brute Force
- GCD Using Euclidean Algorithm
- Prime Number Check variants
- Reverse a Number
- Sum of Digits
- Fibonacci Number
- Decimal to Binary
- Binary to Decimal
- Armstrong Number
- Boston Number
- Chewbacca Number
- Inverse of Number
- Convert Any Base to Any Base
- LeetCode 7
- LeetCode 9
- LeetCode 1492
- LeetCode 202
- LeetCode 3658

Outcome:
- You should be able to recognize when a problem is really about digits, factors, or repeated reduction.

- Normalize input when spaces or punctuation matter
- Think in terms of characters, words, or substrings depending on the task
